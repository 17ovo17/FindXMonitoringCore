## 一、系统概述

### 1.1 项目定位与核心能力

**AI WorkBench** 是一个面向运维工程师的 AI 驱动智能运维平台，具备 7 大核心能力：

1. **智能诊断**：基于 LLM + 工作流引擎 + 结构化推理引擎，对服务器、容器、数据库等进行自动化故障诊断
2. **知识沉淀**：案例库 + Runbook + 文档管理 + 混合搜索，形成可检索、可复用的运维知识体系
3. **告警闭环**：从告警接入到智能路由、自动诊断、报告归档、知识沉淀的全链路闭环
4. **工作流编排**：原生 Go 实现的 DAG 工作流引擎，支持 20 种节点类型、30 个内置工作流
5. **时序分析**：Go 原生时序分析引擎（CUSUM 突变检测 + 趋势回归 + 异常评分 + 周期性检测 + 相关性分析）
6. **混合搜索**：BM25 + 向量搜索 + RRF 融合 + Reranker 的多层检索架构
7. **监控集成**：Prometheus 指标扫描、AI 自动适配、标准名映射、跨数据源关联

## 三、后端架构

### 3.1 分层架构

```
HTTP 请求
    |
    v
[Gin Router + Middleware]  -- RateLimit(100 req/s) + CORS + AdminToken
    |
    v
[Handler 层]               -- 请求解析、参数校验、响应格式化（48 个文件）
    |
    v
[业务引擎层]               -- Workflow / Reasoning / Timeseries / Correlator / EventBus
    |
    v
[Store 层]                 -- MySQL 持久化 + 内存 Fallback + Redis 缓存
    |
    v
[外部集成]                 -- Prometheus / AI Provider / CatPaw Agent / N9e
```

### 4.2 DAG 执行模型

引擎核心位于 `workflow/engine/engine.go`：

- **拓扑排序**：Kahn 算法，检测环路
- **图验证**：唯一 start 节点、至少一个 end 节点、全节点可达性检查
- **超时控制**：工作流级 120s + 节点级 30s（可配置）
- **指数退避重试**：默认 1 次重试，退避 2s 起步，上限 30s
- **并行组执行**：同 `parallel_group` 标记的节点并行执行，WaitGroup 同步
- **Fallback 降级**：节点配置 `on_error: continue` 时返回 fallback_value 继续执行

## 五、结构化推理引擎

### 5.1 假设-验证循环模型

`reasoning/engine.go` 实现了多轮结构化推理：

```
初始数据采集 → 假设生成 → 验证计划执行 → 假设评估 → (循环或终止)
```

- **最大轮次**：`MaxRounds = 3`
- **置信度阈值**：`ConfidenceThreshold = 0.8`
- 每轮循环：为每个 pending 假设执行验证计划 → LLM 评估假设 → 更新置信度
- 当最佳假设置信度 >= 0.8 时提前终止
- 输出包含：根因、分类、置信度、证据链、假设列表、处置方案（即时+永久）

### 5.2 工具注册表

| 工具名 | 功能 |
|--------|------|
| `prometheus_query` | 查询目标主机 Prometheus 指标 |
| `check_alerts` | 查询目标主机活跃告警 |
| `check_diagnose_history` | 查询目标主机最近 5 条诊断记录 |
| `knowledge_search` | 知识库语义搜索 |

支持通过 `Register()` 扩展自定义工具。

### 5.3 数据结构

- `DiagnosticInput`：目标 IP、问题描述、告警标题、严重级别、初始数据
- `Hypothesis`：假设描述、分类、置信度、状态、验证计划、证据列表
- `Evidence`：来源、数据、摘要、支持的假设 ID
- `Treatment`：即时处置 + 永久修复方案

## 六、时序分析引擎

### 6.1 模块概览

`timeseries/` 包含 7 个分析模块：

| 文件 | 功能 |
|------|------|
| `analyzer.go` | 分析编排器，调用各子模块生成综合报告 |
| `changepoint.go` | CUSUM 突变检测 |
| `trend.go` | 线性回归趋势检测 + 预测 |
| `anomaly.go` | Z-score + IQR 异常检测 |
| `seasonality.go` | 自相关周期性检测 |
| `correlation.go` | Pearson 相关性分析 |
| `stats.go` | 基础统计函数（Mean, Std, Percentile, Median, Sigmoid） |

### 6.2 CUSUM 突变检测

`changepoint.go` 实现累积和（CUSUM）算法：

- 计算每个数据点的 Z-score
- 维护正向/负向累积和
- 当累积和超过 `threshold * 3` 时标记突变点
- 输出：突变点索引、方向（increase/decrease）、幅度

### 6.3 线性回归趋势

`trend.go` 使用最小二乘法：

- 计算斜率 slope
- 归一化斜率 `slope / mean`：> 0.01 为 increasing，< -0.01 为 decreasing
- `LinearForecast()` 基于斜率外推未来 N 个数据点

### 6.4 异常检测

`anomaly.go` 提供两种算法：

- **Z-score**：`(latest - mean) / std`，经 Sigmoid 映射到 [0, 1]
- **IQR**：基于四分位距，超出 `Q1 - 1.5*IQR` 或 `Q3 + 1.5*IQR` 判定异常

### 6.5 周期性检测

`seasonality.go` 使用自相关函数：

- 候选周期：6h、12h、24h、168h（一周）
- 要求数据量 >= 48 个点
- 自相关系数 > 0.5 时判定存在周期性

### 6.6 Pearson 相关性

`correlation.go` 计算两组时序数据的 Pearson 相关系数，用于跨指标关联分析。

### 11.2 命令安全分级

`security/guard.go` 实现 5 级安全分类：

| 级别 | 含义 | 处理 |
|------|------|------|
| L0 | 安全 | 放行 |
| L1 | 低风险 | 放行 |
| L2 | 中风险 | 需确认（工作流模式降级为 L1） |
| L3 | 高风险 | 拒绝 |

- PowerShell `-EncodedCommand` Base64 解码后再检查

`middleware/circuitbreaker.go` 提供通用熔断器实现：
