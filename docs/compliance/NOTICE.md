## 项目主线声明

FindX Monitoring Core 是 FindX 项目的新一代监控核心平台，不是 Nightingale 外壳，也不是旧 Catpaw/N9E 入口换名。Nightingale、Categraf、Catpaw 在本项目中的定位分别是：

| 来源 | 当前定位 | 合规口径 |
|------|---------|---------|
| Nightingale | Apache-2.0 参考实现与可融合来源 | 可参考和融合符合许可证要求的设计/源码；正式分发前必须补具体版本、文件清单、修改说明和 NOTICE |
| Categraf | MIT 插件生态复用来源 | 可复用插件生态进入 `findx-agents`；正式分发前必须保留 MIT 许可证声明和来源版本 |
| Catpaw | 已获授权能力的衍生来源 | 只在授权范围内衍生巡检、诊断、会话、结构化工具和自动修复执行能力；授权记录不得公开写入敏感细节 |

## 正式分发前门禁

正式发布、对外交付、二进制分发或容器镜像分发前，必须补齐：

1. 第三方源码来源版本：仓库、tag/commit、下载时间、校验摘要。
2. 修改说明：每个融合文件的改动摘要、保留/删除/重写边界。
3. 许可证文本：Apache-2.0、MIT 和其他实际引入许可证。
4. NOTICE 合并：保留上游要求保留的 copyright、NOTICE 和 attribution。
5. 授权记录：Catpaw 授权范围、限制、期限、责任人和保管位置；公开版本只写 `<AUTH_RECORD>` 占位。
6. 敏感信息复查：不得包含 `<TOKEN>`、`<API_KEY>`、`<DB_DSN>` 的真实值、Cookie、SSH 私钥、平台内网地址或个人敏感信息。

## 禁止事项

- 禁止把 Nightingale 描述为 FindX 的运行时外壳。
- 禁止把 Catpaw 旧接口描述为 FindX 新主线。
- 禁止在未记录来源版本和许可证义务前进行正式分发。
- 禁止把授权内容、密钥、Cookie、完整连接串或内部平台地址写进公开文档。
