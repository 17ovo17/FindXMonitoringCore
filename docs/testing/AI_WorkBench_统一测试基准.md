## 第一章 总则与执行规范

> 定义测试执行的安全边界、命令分级体系、禁止清单及商业可用评分标准，是所有后续测试章节的强制前置约束。

### 1.1 Codex 子代理默认测试基准

本节是 `qa-tester` 子代理的默认执行口径。任何 AI WorkBench 相关测试、修复回归、发布前验收，都必须优先引用本文档中的用例编号；新增功能尚无编号时，临时编号使用 `QA-{层}-{模块}-{序号}`，例如 `QA-API-ALERT-001`、`QA-UI-TOPOLOGY-003`、`QA-REG-LOGIN-001`。历史 `COD-QA-*` 编号可继续识别，但新增用例统一使用 `QA-*`。

#### 1.1.1 结果状态

| 状态 | 含义 | 是否可作为通过 |
|------|------|----------------|
| PASS | 已按步骤真实执行，结果与预期一致，证据完整 | 是 |
| FAIL | 已执行但结果不符合预期，或发现 P0/P1/P2/P3 缺陷 | 否 |
| BLOCKED | 因环境、依赖、权限、服务不可用或安全边界无法执行 | 否 |
| NOT_RUN | 本轮未执行，原因和补测入口已说明 | 否 |
| RISK | 功能可用但存在架构漂移、技术债、证据不足、可观测性缺口或发布风险 | 否，除非主代理明确接受风险 |

历史文档中的 `NOT_TESTED` 与 `NOT_RUN` 等价；新报告统一写 `NOT_RUN`。未验证项不得写成 PASS；API 通过不能替代 UI/MCP 真实交互通过；开发代理自测不能替代 `qa-tester` 或主代理回归结论。P0/P1 级 RISK 默认阻断，只有主代理明确接受并记录风险时才允许有条件通过。

#### 1.1.2 优先级定义

| 优先级 | 定义 | 默认处理 |
|--------|------|----------|
| P0 | 阻断：安全漏洞、数据丢失、核心链路不可用、敏感信息泄露、权限绕过、L4 危险命令未拦截 | 禁止通过，必须修复 |
| P1 | 高：关键功能不可完成、主要 API/UI/任务异常、降级不可用、迁移不可回滚 | 禁止通过，必须修复 |
| P2 | 中：非核心功能异常、体验问题、性能或兼容问题 | 可遗留，但必须记录风险、责任和计划 |
| P3 | 低：增强建议、轻微体验或文档优化 | 可遗留，进入待办 |

#### 1.1.3 测试编号

新增临时用例统一使用 `QA-{层}-{模块}-{序号}`：

| 层 | 含义 |
|----|------|
| API | 接口、认证、响应、错误码、契约兼容 |
| UI | 页面、组件、按钮、表单、弹窗、浏览器交互 |
| DATA | 数据库、迁移、事务、缓存、备份恢复 |
| TASK | 定时任务、后台任务、队列、重试、并发 |
| DOC | README、API 文档、运维手册、测试文档 |
| SEC | 注入、越权、命令分级、白名单、脱敏 |
| DEG | 依赖不可用、fallback、恢复后重连 |
| REG | 旧功能、旧接口、旧数据和导航回归 |
| PERF | 性能、压力、资源消耗 |
| OPS | 诊断、Runbook、指标、告警、值班闭环 |

#### 1.1.5 回归选择规则

- 改 API：回归所有前端调用点、curl 契约、认证、错误码、旧响应兼容和文档同步。
- 改 UI：回归对应页面、可点击元素、表单、loading/error/empty 状态、浏览器前进后退、响应式和控制台错误。
- 改数据表/迁移：回归旧数据读取、迁移、回滚、备份恢复、幂等、数据清理和受影响接口/页面。
- 改认证/权限：回归未登录、过期登录态、低权限、水平越权、垂直越权、401/403 脱敏。
- 改后台任务/调度：回归定时触发、手动触发、并发、重复执行、失败重试、审计和恢复。
- 改配置/外部依赖：回归配置默认值、旧配置兼容、依赖不可用降级、恢复后重连和日志脱敏。
- 改文档：回归命令、路径、端口、接口、配置项、截图/示例是否与代码和脚本一致。

#### 1.1.6 每批测试最小矩阵

每个 `qa-tester` 批次至少输出以下矩阵。不能执行的项目必须标为 BLOCKED 或 NOT_RUN，并说明补测入口。

| 维度 | 必填内容 |
|------|----------|
| 范围 | 本轮变更文件、功能点、API、页面、数据流、依赖服务 |
| 用例映射 | 引用本文档用例编号；新增临时用例使用 `QA-*` |
| 风险分级 | P0/P1/P2/P3；安全、数据、权限、发布阻断必须 P0/P1 |
| 路径覆盖 | 正常、异常、边界、权限、安全、降级、回归 |
| 分层覆盖 | API、UI、数据、任务/调度、AI/工作流、文档 |
| 结果状态 | PASS、FAIL、BLOCKED、NOT_RUN、RISK |
| 证据 | 命令、响应摘要、截图、快照、日志摘要、pre/post snapshot、teardown |
| 归属 | `go-backend`、`vue-frontend`、`ops-diagnostician`、`doc-writer` 或 `main-agent` |

#### 1.1.7 单条测试记录字段

| 字段 | 要求 |
|------|------|
| case_id | 统一基准编号或 `QA-*` 临时编号 |
| title | 测试点标题 |
| layer | API/UI/DATA/TASK/DOC/SEC/DEG/REG/PERF/OPS |
| scenario | 正常/异常/边界/权限/安全/降级/回归 |
| priority | P0/P1/P2/P3 |
| status | PASS/FAIL/BLOCKED/NOT_RUN/RISK |
| steps | 可复现步骤 |

#### 1.1.8 阻断条件

出现以下任一情况，`qa-tester` 必须判定本批次 FAIL 或 BLOCKED，不允许给出通过结论：

- P0/P1 缺陷未修复。
- S0 安全预检失败，或白名单外副作用无法确认。
- 敏感信息明文出现在页面、日志、响应、文档、截图或测试报告中。
- P0/P1 用例缺少证据文件。
- API_CONTRACT_CHANGE 或 DATA_CHANGE 未打标、未验证兼容或未说明回滚。
- 写操作未覆盖重复提交、非法输入、权限失败。
- UI 核心链路未做 MCP/Playwright 真实交互，只有 API 证据。
- 测试数据未隔离或 teardown 未执行。
- 子代理发现 P0/P1 架构漂移、技术债或未验证风险且主代理未裁决。

## FindX 2026-05 强制准出门禁

### MCP 浏览器真实登录必测

- 所有 UI、导航、登录态、权限、表单、抽屉、弹窗、导入导出、查询、告警、通知、Agent、AI SRE 相关变更，必须使用 MCP/Playwright 进行真实浏览器登录和点击回归。
- API 测试、curl、静态截图、DOM 结构检查只能作为辅助证据，不能替代真实浏览器操作。
- 登录失败、登录过期、401/403、组件不可用、路由跳转异常必须覆盖；登录页不可被参考站 SSO 或外部系统嵌入替代。
- 回归记录必须包含 URL、账号占位符、操作步骤、实际结果、截图或 DOM 证据、状态 PASS/FAIL/BLOCKED/NOT_RUN/RISK。

### Agent 生命周期必测

- Agent 管理相关变更必须覆盖安装计划、凭据引用、远程执行、包校验、服务注册、配置下发、心跳上报、数据到达、异常恢复、升级、回滚、卸载、审计和 Evidence Chain。
- Linux 必测 SSH/shell/systemd/Docker 场景中的适用项；Windows 必测 WinRM/PowerShell/Windows Service/IIS/.NET 场景中的适用项；Kubernetes 必测 Helm/Operator/Webhook/DaemonSet/Sidecar/InitContainer/RBAC/TLS 场景中的适用项。
- 无法访问真实环境时必须标记 BLOCKED 或 NOT_RUN，并说明缺少的环境、凭据、组件或数据，不得用静态页面冒充完成。

### 组件不可用必须 BLOCKED

- SkyWalking OAP、BanyanDB、SigNoZ、ClickHouse、OTel Collector、Qdrant、Prometheus/VictoriaMetrics、MariaDB、Redis、MinIO、远程执行通道等组件不可用时，页面和 API 必须显示明确 BLOCKED 或 503 脱敏错误。
- 不得伪造成功状态、假数据、静态列表、空壳图表或“演示成功”。
- BLOCKED 记录必须写明组件名、期望配置、探测方式、实际错误和恢复建议，不能泄露 token、cookie、DSN、私钥或完整连接串。

### 禁止静态假按钮和未验证 PASS

- 成熟源码中具备真实动作的按钮、下拉、搜索、联想、历史、添加、导入、分享、放大、设置、三点菜单、启停、克隆、删除、批量操作，在 FindX 中必须接入真实动作。
- 未接入真实动作时状态只能是 FAIL 或 BLOCKED，不能写 PASS。
- 不接受“最小实现”“最小验证”“先占位”“后续补齐”作为验收结论。
- P0/P1 RISK、敏感信息、权限绕过、错误未脱敏、外部品牌用户侧暴露、乱码、未跑 MCP 浏览器回归均阻断通过。
