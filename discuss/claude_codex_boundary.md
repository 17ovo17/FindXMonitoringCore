# Claude 不在时的强制边界

当 Claude 不在场时，主代理只临时代行编排、审计、验收、回派和 Git 门禁职责，不直接修改 `api`、`web`、`docs` 等业务或产品文件。主代理可以读代码、查证据、拆分工作单元、准备验证命令、同步 WSL、执行构建和回归，并基于证据给出通过、退回或需讨论的门禁结论，但不得顺手替写业务逻辑。

子代理是唯一执行层。编码、文档、测试、页面修复、接口修复和诊断设计等具体改动，必须派发给对应子代理完成；派发内容必须自包含任务目标、允许路径、禁止路径、验收标准、验证命令和敏感信息占位符规则。同一批并行子代理不得修改同一文件或同一不可分割模块。

子代理卡住、超时或返回不完整时，主代理只能继续做只读审计、验证准备、风险检查、重新派发或回派修正，不得以“临时补一下”为由接管业务代码。确需主代理临时接管时，必须先在 `.claude/operations-log.md` 记录原因、范围、风险、允许路径和回滚方式，再按已记录范围执行。

任何前端、后端、文档变更都必须先读再写、先搜再建、先拆再做：修改前理解现有结构和风格，新建前搜索可复用实现，大任务拆成可验证的小步骤。不得用文档替代代码验证，不得留下占位实现或未说明的并行旧入口。

提交前，乱码、权限绕过、敏感信息泄露和 Git 门禁失败均为阻断项。发现真实密钥、认证票据、Cookie、完整连接串、会话 ID，或未脱敏的内部敏感信息时，必须停止提交并回派修正。QA FAIL、P0/P1 缺陷、P0/P1 风险未关闭时，不得进入 Git 门禁。
